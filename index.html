浅谈Minecraft 致我的10年Minecraft时光

上篇 经历

第一阶段 4~8岁

那个时候还在玩迷你世界，也正值互联网一战爆发，我也参与了其中。最开始也是坚定的支持迷你世界，后来了解到MC才是正统，转头直接“弃暗投明”，转战MC。中间还当了一段时间两面派，即既支持MC又支持迷你，主张两边和平共同发展，然后不出所料，也是被骂了好吧。

第二阶段 8岁~2022年

由于要学编程，我爸给我配了电脑，剧情开始反转。

我第一次接触了Java版MC，再也不用受那个神人网易的恶心，也体验了JAVA版那难以置信的自由度，正所谓喜新厌旧，扔掉网易账号直接不玩了。

同时还有一个支线，——骂迷你。当时“一战”正酣，我和我的一个朋友每天放学第一件事就是在回家的路上控诉迷你有多么多么烂，多么多么能抄，玩家素质有多么多么低下。

2022年11 月 30 日，据广东省高级人民法院消息，广东省高级人民法院对广州网易公司、上海网之易公司诉深圳迷你玩公司著作权侵权及不正当竞争纠纷案作出终审判决，认定深圳迷你玩公司构成不正当竞争，判令其删除游戏中 230 个侵权元素，并赔偿网易公司 5000 万元。据了解，这是国内游戏侵权纠纷案件最高判赔数额。2017 年 8 月 15 日，网易以涉嫌抄袭其代理的游戏《我的世界》为由对迷你玩发出维权函，并要求进行删除下架处理。2018 年至 2021 年间，网易和迷你玩多次对簿公堂，至此，持续近四年的中国沙盒游戏纠纷案尘埃落定。

一战结束了，什么都结束了。大家狂欢了一段时间，后来这件事也就不了了之了。

第三阶段 2023年至今

我父亲给我换了台新电脑（后话，应该在2024年年底换的），用上4080S了，曾经那些打不开的整合包，那些跑不动的光影，终于能打开了，并且游刃有余。我也更加深度的体验了各种游戏内容，可是，然后呢？
太空虚了，说实话，体验完一点刺激感都没有，那种新奇，早就在数年，甚至说十年的时间中消磨殆尽了。于是，我下载了GTNH，准备用几年时间专心玩这一个整合。

下篇 About Minecraft Editions

1.7.10

让我们从1.7.10开始。

1.7.10，第一个模组大爆发的版本。

起因是因为Mojang在1.7这一个大版本中更了10个小版本，导致玩家社区对新内容的需求暴增，于是模组社区便开始蓬勃发展。像我们所熟知的等价交换，无尽贪婪，匠魂，虚无世界，拔刀剑，暮色森林等老牌模组都是这个时期的产物，这些模组也构成了我们的回忆（至少是我的）。

###1.8.9

这个版本要特别提一下，因为下一个版本（1.9.0）改革了攻击机制，加入了攻击冷却。这个版本也是一个争议极大的版本，无数服务器永远停留在了1.8.9，就是因为攻击冷却（主要为PVP服务器）。

1.12.2

又一个模组大爆发的时代，这次是因为1.13Mojang做了一个叫扁平化的工作，导致模组社区大受打击，以至于Forge拖了大半年才更新。这么长的时间也给了模组社区发展的窗口。现在大家所熟知的RLcraft，Skyfactory，GreedyCraft等整合包全都是来自于这个版本，至此，低版本阶段正式结束。低版本，经典中的经典。

1.13~1.14

水域更新与村庄更新（正式名称村庄与掠夺）。1.13丰富了水域的生态，珊瑚扇为未来的TNT复制与TNT科技铺垫了基础。1.14改革了村庄生成，村民交易机制，把原本拉完了的村庄升级为夯中夯的结构，村民正式成为香饽饽，这个版本也是高版本生电的起点。

1.15

嗡嗡蜂群。这个版本加入了蜜蜂机制，同时大幅对游戏进行了优化，为接下来的一个版本做足了铺垫。

1.16

下界更新。毋庸置疑的夯中夯中夯。继1.12后又一个模组大爆发的时代，模组数量同时超过1.7.10与1.12.2的一个版本（1.16.5）。下界更新丰富了下界的生物群系，创造了一个速通的黄金版本——1.16.1，加入了一个新结构猪灵堡垒，加入了猪灵交易机制（？！强强！？），加入了下界合金装备与武器，极大的加强了下界的可玩性，使得下界成为了一个极为优秀的维度。机械动力——一个划时代的模组，将现代科技模组带进了新时代，不再向以前的超现实科技模组学习（AE2，IC2，EnderIO，HBM，沉浸工程等），为后续高版本科技模组大爆发奠定了基石。

1.17~1.18

洞穴与山崖Part1&Part2。一个拖更了许久的版本。1.17主要加物品，1.18主要改地形。这两个版本如果拼到一起，也是一个相当相当不错的版本，可惜了。

1.19

洞穴与山崖Part3荒野更新。加入了坚守者和远古城市。还加入了带坏无数模组的锻造模板，总体上讲，这个版本利大于弊（主要古城太肥了）。

1.20

洞穴与山崖Part4足迹与故事。加入了考古（不是重点）和樱花树林（这个很重要），1.20.4与1.20.1都是模组大爆发的版本，总体上是Fabric蓬勃发展，元素周期表那一套对游戏的优化是真可以。

1.21

洞穴与山崖Part5棘巧试炼。加入试炼大厅和涂蜡的氧化切制铜楼梯和乐魂（肯定还有其他东西），1.21.4还把刷线机修了（这个倒没毛病，这玩意太逆天了），1.21.1又有很多新模组，比如航空学，瓦基（这个应该要早一点），还有一大票老模组复活，真的让人热泪盈眶啊。小乐魂是真可爱，大乐魂有点用但不多，主要是可以高空作业了（乐魂战除外）。

C++/cpp入门教程

2026-07-10

本教程默认读者已经装好编译器，编译器（VS2022 MSVC）配置教程择期发布。

#include <iostream> int main(){ std::cout<<"Hello World!"<<endl; return 0; }

让我们从第一行开始。

第一行：#include <iostream>

include，意为导入，其意义便是导入头文件。
头文件是大部分C++程序运行时不可或缺的东西，接下来，例如vector，algorithm，map，queue等头文件会时常被我们提起。
请注意，许多教程都会提到一个叫做bit/stdc++.h的东西，这个东西常被称为万能头文件。但在我们讲解中使用的MSVC编译器中没有该头文件，强制引用会报错。

第二行：int main()

这是C++程序运行的主函数，它不可以改名，不可以不写，并且在初学阶段我强烈不推荐任何人使用void main，signed main之类的写法。

第三行 std::cout<<”Hello World!”<<endl;

std::是标准命名空间前缀，今后想参加大型程序开发的各位，一定不要偷懒去写什么using namespace std;，没有std:: 很容易报各种莫名其妙的错误。
cout是输出流函数，用于输出（想用C风格输出欢迎使用printf）。

表示输出流，在今后的字符串操作中我们会详细讲到它
“Hello World!” 表示一个内容为 Hello World! 的字符串，同样，对于此类字符串操作，我们同样在后面会讲到它。
注意，绝对不要写成’’单引号

第五行 return 0;

返回值，默认返回0表示程序运行成功（并不代表没有bug）

第一期到此结束
